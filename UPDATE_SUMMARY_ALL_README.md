## ⚙️ 更新的字段列表

### 基本汇总字段

| 字段 | 说明 |
|------|------|
| `overview_total_energy` | 总用电量 (kWh) |
| `overview_total_cost` | 总电费 (元) |
| `overview_poi_count` | POI 总数 |
| `overview_device_count` | 电表总数 |
| `record_count` | 原始记录数 |

### 用电方维度字段

| 字段 | 说明 |
|------|------|
| `mobile_cumulative_energy` | 移动用户日累计用电量 (kWh) |
| `mobile_poi_count` | 移动用户 POI 数量 |
| `tower_cumulative_energy` | 电塔用户日累计用电量 (kWh) |
| `tower_poi_count` | 电塔用户 POI 数量 |

### 供电类型维度字段

| 字段 | 说明 |
|------|------|
| `direct_power_supply_energy` | 直供电用电量 (kWh) |
| `direct_power_supply_cost` | 直供电电费 (元) |
| `indirect_power_supply_energy` | 转供电用电量 (kWh) |
| `indirect_power_supply_cost` | 转供电电费 (元) |

### 电费字段

| 字段 | 说明 |
|------|------|
| `mobile_electricity_fee` | 移动用户日累计电费 (元) |
| `tower_electricity_fee` | 铁塔用户日累计电费 (元) |

## 🔄 处理逻辑

### 单日更新流程

```
1. 检查源表中是否存在该日期的数据
2. 删除目标表中该日期的旧数据
3. 插入新的基本汇总数据
4. 更新用电方维度字段（移动/电塔）
5. 更新供电类型维度字段（直供电/转供电）
6. 更新电费字段
```

## 🛠️ 常见问题

### Q1: 如何查看所有可用日期？

可以通过数据库查询：

```sql
SELECT DISTINCT 日期 FROM energy_charge ORDER BY 日期 DESC;
```

### Q2: 批量更新可以中断吗？

可以。使用 `Ctrl + C` 中断，但已处理的数据不会回滚。

### Q3: 数据验证失败怎么办？

脚本会自动检查源表数据是否存在。如果失败，检查：
1. 数据库连接是否正常
2. 源表 `energy_charge` 中是否有该日期的数据

### Q4: 批量更新太慢怎么办？

可以分批执行：

```bash
# 分批执行
python update_summary_all.py 2026-01-01
python update_summary_all.py 2026-02-01
python update_summary_all.py 2026-03-01
```
