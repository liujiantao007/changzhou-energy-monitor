# 常州能耗云运营驾驶舱 - 部署文档

## 1. 项目结构

```
project_dianfeiv2/
├── css/
│   └── style.css          # 样式文件
├── js/
│   ├── app.js             # 应用主逻辑
│   ├── data.js            # 数据处理模块
│   ├── charts.js          # 图表模块
│   └── map.js             # 地图模块
├── lib/
│   ├── echarts.min.js     # ECharts库（需下载）
│   ├── xlsx.full.min.js   # SheetJS库（需下载）
│   └── README.md          # 依赖库说明
├── data/
│   ├── data.xlsx          # 能耗数据文件
│   └── 常州市含经开区.json  # 常州地理数据
├── index.html             # 项目入口文件
└── 部署文档.md            # 部署文档
```

## 2. 本地环境搭建

### 2.1 前置要求
- 浏览器：支持IE11+、Chrome、Firefox、Safari等主流现代浏览器
- 网络：首次部署需要互联网连接以下载依赖库
- 硬件：建议使用分辨率1920×1080及以上的显示器

### 2.2 依赖库下载
1. **ECharts库**：用于图表和地图展示
   - 下载地址：https://cdn.jsdelivr.net/npm/echarts@5.4.3/dist/echarts.min.js
   - 保存路径：`lib/echarts.min.js`

2. **SheetJS库**：用于解析Excel文件
   - 下载地址：https://cdn.jsdelivr.net/npm/xlsx@0.18.5/dist/xlsx.full.min.js
   - 保存路径：`lib/xlsx.full.min.js`

### 2.3 数据准备
1. 确保`data`目录下存在`data.xlsx`文件，包含能耗数据
2. 确保`data`目录下存在`常州市含经开区.json`文件，包含常州地理数据

## 3. 内网部署指南

### 3.1 部署步骤
1. 将整个项目目录复制到内网服务器或本地计算机
2. 确保所有依赖库已正确下载并放置在`lib`目录中
3. 确保数据文件已正确放置在`data`目录中
4. 在浏览器中打开`index.html`文件即可访问系统

### 3.2 离线运行说明
- 项目所有资源均为本地引用，无外部CDN依赖
- 系统启动时会自动读取本地Excel文件进行数据初始化
- 无需安装任何服务器软件，直接通过浏览器打开即可运行

### 3.3 分辨率适配
系统支持以下主流大屏分辨率：
- 1920×1080（全高清）
- 2560×1440（2K）
- 3840×2160（4K）

系统会自动根据屏幕分辨率进行等比例缩放，确保无黑边、无滚动条，内容完整显示且不失真。

## 4. 系统维护

### 4.1 数据更新
1. 替换`data/data.xlsx`文件即可更新数据
2. 系统会在下次启动时自动加载新数据

### 4.2 地图数据更新
1. 替换`data/常州市含经开区.json`文件即可更新地图数据
2. 系统会在下次启动时自动加载新地图数据

### 4.3 功能扩展
如需扩展功能，可修改以下文件：
- `js/app.js`：应用主逻辑
- `js/data.js`：数据处理逻辑
- `js/charts.js`：图表配置
- `js/map.js`：地图功能
- `css/style.css`：样式调整

## 5. 故障排查

### 5.1 常见问题
1. **页面空白**：检查依赖库是否正确下载，确保`lib`目录中存在`echarts.min.js`和`xlsx.full.min.js`文件

2. **数据不显示**：检查`data/data.xlsx`文件是否存在，格式是否正确

3. **地图不显示**：检查`data/常州市含经开区.json`文件是否存在，格式是否正确

4. **图表不显示**：检查浏览器控制台是否有错误信息，可能是ECharts库加载失败

### 5.2 浏览器兼容性
- IE11：支持基本功能，但部分动画效果可能不流畅
- Chrome/Firefox/Safari：支持所有功能，效果最佳

## 6. 性能优化建议

1. **数据量较大时**：考虑对Excel数据进行预处理，减少解析时间

2. **地图加载缓慢**：可考虑简化地理数据文件，减少文件大小

3. **页面卡顿**：可关闭浏览器的开发者工具，减少性能消耗

4. **长时间运行**：建议定期刷新页面，避免内存泄漏

## 7. 部署验证

部署完成后，可通过以下步骤验证系统是否正常运行：

1. 打开`index.html`文件
2. 检查页面是否正常加载，无错误提示
3. 检查能耗总览数据是否显示
4. 检查图表是否正常渲染
5. 检查地图是否正常显示
6. 检查导航功能是否正常
7. 检查响应式缩放是否正常（可调整浏览器窗口大小测试）

如所有检查项均通过，则部署成功。
